# Observability (관측성)

시스템의 내부 상태를 외부 출력을 통해 파악할 수 있는 능력

## 3가지 핵심 요소 (Three Pillars)

```
Observability
├── Metrics (메트릭)
├── Logs (로그)
└── Traces (트레이스)
```

### Metrics (메트릭)

- **정의**: 시간에 따른 수치 데이터
- **특징**: 집계 가능, 저장 비용 낮음, 트렌드 파악에 유리
- **예시**: CPU 사용률, 요청 수, 에러율, 응답 시간
- **도구**: Prometheus, Datadog, InfluxDB

### Logs (로그)

- **정의**: 시스템에서 발생한 개별 이벤트 기록
- **특징**: 상세한 컨텍스트 제공, 디버깅에 유리
- **예시**: 에러 메시지, 요청/응답 내용, 애플리케이션 상태 변화
- **도구**: ELK Stack, OpenSearch, Loki

### Traces (트레이스)

- **정의**: 요청이 시스템을 통과하는 전체 경로 추적
- **특징**: 분산 시스템에서 요청 흐름 파악, 병목 지점 식별
- **예시**: API 호출 → 서비스 A → 서비스 B → DB 쿼리
- **도구**: Jaeger, Zipkin, Datadog APM

## 비교

| 구분 | 질문 | 용도 |
|------|------|------|
| Metrics | "얼마나?" | 시스템 상태 모니터링, 알림 |
| Logs | "무슨 일이?" | 디버깅, 감사 |
| Traces | "어디서 느려졌나?" | 성능 분석, 병목 탐지 |
